## Overview

Goal: make Hatchin trustworthy in two dimensions at once — autonomy backend reliability (no runaway spend, scheduled routines) AND conversation experience reliability (Maya knows when to start, agents act like a real team, deliverables ship).

**Why partial close-out:** After Phase 22 (atomic budget) and Phase 28 (Maya bug fix + SDK migration) shipped, an Apr 25–28 deep audit (18 repo evaluations + 15-item gap audit) produced ROADMAP-V3, which **re-scoped** the remaining 11 v3.0 phases across V3's milestone structure (v2.1 + v2.7). No v3.0 work was abandoned — it was re-homed.

## Milestone Summary

**Decisions made during v3.0:**
- Bundle Pillar A (autonomy reliability) + Pillar B (Maya/teamness) into single milestone — same trust signal
- Pattern A atomic ledger (vs serializable transaction) for budget enforcement — proven via 22-01 concurrency test
- `@google/generative-ai` → `@google/genai` migration was mandatory prerequisite for AbortSignal hygiene (deprecated SDK had unresolved Issue #303)
- Phase 22 + 28 ship in parallel (independent dependency chains)

**Issues resolved:**
- BUDG-01..03: budget race condition closed (was at `taskExecutionPipeline.ts:543-560`)
- BUG-01..06: Maya infinite-thinking, stop button stuck, heap leak (70.58MB → 0.11MB), zombie fallback messages
- DB-CRASH-01 (post-Phase 28 hotfix): Neon idle-in-transaction recovery + traceStore transaction leak

**Issues deferred (to V3 roadmap):**
- All 11 unfinished phases re-scoped to V3 milestones (v2.1 + v2.7 + v3.0 + v2.3) — see per-phase re-home tables above

**Technical debt incurred:**
- `wip/pre-reset-2026-04-28` branch contains uncommitted UI experimentation (`ProjectTree.tsx` 5-line deletion) — addressed separately
- ROADMAP.md and REQUIREMENTS.md were single-milestone-scoped; this close-out archives them

**Audit verdict:** v3.0 close-out is paperwork-only. No code gaps to ship. STATE.md was stale ("91% complete, 22-03 pending") — the 91% number conflated "shipped phases" with "phases acknowledged in V3."
