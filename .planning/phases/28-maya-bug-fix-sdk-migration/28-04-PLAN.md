<objective>
Close BUG-03 (thinking-state stuck after timeout) and BUG-04 (zombie "out for lunch" message arriving late after timeout). Both bugs live in `server/routes/chat.ts` catch-blocks within ~2000 lines of each other and have a shared root cause: the outer Promise.race fires at 60s, sends `streaming_error`, but the inner async function continues running and eventually sends a fallback message over the still-open WS.

Purpose: After plan 28-02 lands, the SDK call respects the 30s AbortSignal and throws AbortError/TimeoutError up through the stream generator. This plan ensures (a) the client's typing indicator clears on the error path, and (b) no zombie fallback fires when the request was already aborted.

Output: Two narrow edits to `server/routes/chat.ts`. After this plan: a hung Gemini request triggers `typing_stopped` + `streaming_error` at 30s, and no spurious "out for lunch" appears in the chat.
</objective>
