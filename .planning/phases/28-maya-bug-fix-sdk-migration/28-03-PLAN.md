<objective>
Fix BUG-06: stop button stuck after a Hatch finishes responding. Two surgical client-side changes — reduce the truth-enforcer debounce from 300ms to 0ms (microtask tick) and force `metadata.isStreaming: false` when constructing the persisted message in the `new_message`/`chat_message` handler.

Purpose: User testing confirmed the stop button stays in "Stop" state after `streaming_completed` (success path). Research (28-RESEARCH.md lines 244-323) traced this to a React batching race in the truth-enforcer — `setIsStreaming(false)` fires AFTER `setAllMessages` updates, so for one render cycle the derived `isStreaming` prop sees stale state. The 300ms debounce on cleanup is also visible to the user.

Output: One file modified (`client/src/components/CenterPanel.tsx`) with two narrow edits. After this plan: clicking send → response arrives → button shows "Send" within 1 second of the last assistant chunk landing in the DOM.

This plan is INDEPENDENT of the SDK migration (28-02) and can run in parallel — only client-side code is touched.

```typescript
// === Streaming state truth-enforcer ===
// isStreaming=true must imply a placeholder message with status='streaming' exists.
// Events can be lost (mobile flaky WS, server emitting chat_message instead of
// streaming_completed). If the placeholder is gone but isStreaming is still true,
// clear it — the stop button has no valid thing to stop.
useEffect(() => {
  if (!streaming.isStreaming) return;
  const convId = currentChatContext?.conversationId;
  if (!convId) return;
  const msgs = messages.allMessages[convId] || [];
  const hasLiveStream = msgs.some(m =>
    m.status === 'streaming' ||
    (m.metadata && (m.metadata as any).isStreaming === true)
  );
  if (hasLiveStream) return;
  // Give React a tick to settle — streaming_started sets isStreaming(true) and
  // adds the placeholder in the same pass; the effect then sees both.
  const t = window.setTimeout(() => {
    streaming.setIsStreaming(false);
    streaming.streamingMessageId.current = null;
  }, 300);   // ← change to 0
  return () => window.clearTimeout(t);
}, [messages.allMessages, currentChatContext?.conversationId, streaming.isStreaming]);
```
